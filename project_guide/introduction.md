### 1. 프로젝트 명

**LegisGuard AI**

> 
> 입법예고(텍스트 / JSON / PDF)를 입력으로 받아 사전 구축된 법령 구조 DB 및 Vector DB 기반 지식 레이어와 비교 분석하고,
> 
> 변경 전·후 차이 및 조직의 대응 액션을 도출하는 **멀티 에이전트 기반 법규 영향 분석 시스템**
> 

### 2. 배경 및 문제 정의

- **현재 상황 (As-Is):**
    
    기업의 안전·보건·환경·시설 관련 부서는 국민참여입법센터, 법제처, 관보, 각 부처 홈페이지 등을 개별적으로 확인해야 한다.
    
    특히 다음과 같은 문제가 반복된다.
    
    - 입법예고 단계에서 변경 내용을 조기에 인지하지 못함
    - 법령 개정 전·후 조문 비교를 수작업으로 수행
    - 개정 영향 범위를 내부 규정·업무 프로세스와 매핑하지 못함
    - 공포·시행 이후에야 대응하며 리스크 발생
    
    입법예고는 공개되어 있으나, **“변경의 의미”와 “우리 조직에 필요한 조치”로 자동 연결되지 않는다.**
    
    또한 대부분의 조직은 법령 데이터를 체계적으로 구조화·정규화해 보유하지 않아, 비교 분석 자체가 반복적·수작업 중심으로 수행되고 있다.
    
- **핵심 문제 정의**
    
    > ***“입법예고 및 법령 개정 정보는 공개되어 있으나, 이를 기존 법령과 비교하여 조직의 구체적 대응 액션으로 연결하는 과정이 수작업·전문가 의존 구조로 남아 있어 대응 지연과 리스크가 발생한다.”***
    > 
- **Pain Point**
    - **분석 시간 손실**
        - 개정안 vs 현행법 조문 대조 수작업
        - PDF 기반 비교 시 가독성 저하
        - 평균 수 시간~수일 소요
    - **리스크 노출**
        - 시행 이후 내부 규정 미정비
        - 과태료·형사처벌·행정처분 가능성
        - 중대재해·산업안전 관련 법령의 경우 직접적 법적 책임 발생 가능
    - **전문성 의존**
        - 법무팀 또는 외부 자문 의존
        - 조직 전체의 법령 이해도 편차
- **기회 요인**
    - 국민참여입법센터에서 입법예고 데이터 공개
    - 법령은 조문 단위의 구조화가 가능한 문서 체계
    - LLM + RAG 기반 조문 의미 분석 가능

### 3. 목표 사용자 (Target User)

- **안전보건팀**
    - 산업안전보건법, 중대재해처벌법 대응
- **환경안전(EHS) 담당자**
    - 화학물질, 폐기물, 실내공기질, 환경보건 관련 대응
- **법무팀**
    - 개정 리스크 분석 및 대응 전략 수립
- **경영진**
    - 법적 리스크 요약 보고 필요

### 4. 프로젝트 목표

- 정성적 목표
    - 입법예고 → 변경 분석 → 영향 도출 → 액션 제안까지 자동화
    - 조문 단위 정밀 비교
    - 조직 적용 관점의 실행 가능한 인사이트 제공
- 정량적 목표 (KPI)
    
    
    | 성과 지표 | 목표 | 측정 방법 |
    | --- | --- | --- |
    | 입법예고 분석 처리 시간 | 평균 5분 이내 | 입력 시점 ~ 최종 리포트 생성까지 자동 측정 |
    | 조문 매칭 정확도 | 95% 이상 | 사전 정의 테스트 세트 기준 매칭 성공률 |
    | 조문 변경 탐지 재현율 | 95% 이상 | 변경 조문 기준 탐지 성공률 |
    | 변경 유형 분류 정확도 | 90% 이상 | 사전 라벨 세트와 비교 |
    | 조직 적용성 판단 정확도 | 90% 이상 | 수동 평가 대비 일치율 |
    | 액션 제안 생성율 | 100% | 변경 발생 시 최소 1개 이상 제안 생성 |

### 5. PoC 범위 정의

**기본 시나리오**

> “입법예고 1건을 입력받아 영향 분석 수행”
> 

**대상 범위**

- 한국 법령 한정
- 일부개정안 중심
- 안전·보건·환경 분야 법령(사전에 정의)

**기능 범위**

- 입법예고 문서 파싱 (텍스트 / JSON / PDF)
- 입법예고 유형 자동 분류 (일부개정 / 전부개정 / 신설 등)
- 사전 구축된 법령 구조 DB 조회
- 조문 단위 Diff 분석
- 변경 유형 분류
- 조직 적용성 분석
- 액션 아이템 제안

**제한 사항**

- 법률 자문 대체 목적 아님
- 한국 법령 중심 PoC

### 6. 데이터 소스 및 접근 방식

- **국민참여입법센터**
    - 입법예고 목록 수집
    - 상태: 공포·시행 단계 모니터링
- **법제처 Open API**
    - 현행 법령 조문 단위 조회
    - 시행령·시행규칙 포함
    - 구조화된 XML/JSON 수집
- **법령 구조 DB (Structured Law DB)**
    - 법 → 장 → 절 → 조 계층 구조 저장
    - 조문 ID 표준화
    - 최신 시행 기준 반영
    - RDB 기반 저장
- **법령 Vector DB**
    - 저장 단위: 조문 단위
    - 메타데이터: 법령ID, 조문번호, 시행일
    - 임베딩 기반 의미 검색 지원
    - RAG 기반 의미 해석 지원
- **조직 내부 매핑 데이터 (임시 데이터 생성)**
    - 사업장 유형
    - 설비 유형
    - 위험물 취급 여부
    - 보건관리 체계 등
- **RAG 기반 전문 지식**
    - 법제처 Open API 로 기존에 수집한 법규 현행 데이터

- **Multi-Agent 구조**
    
    기능 단위가 아닌 **관점(Reasoning Perspective)** 기반 책임 분리
    
    - **Orchestrator**
        
        **역할**
        
        - 입법예고 입력 수신
        - 개정 / 신설 / 삭제 유형 판단
        - Agent 호출 전략 분기
            - 개정 / 삭제인 경우 Change Analyst Mode : `DIFF`
            - 신설인 경우 Change Analyst Mode : `STRUCTURE`
        - 각 Agent 결과 수집 및 종합
        - 최종 리포트 생성
        
        **특징**
        
        - 추론은 최소화
        - 워크플로우 제어 중심
        - 상태 관리 담당
    - **Change Analyst (법령 구조 전문가)**
        
        **역할**
        
        - 조문 단위 diff 생성
        - 신설 / 삭제 / 변경 부분 식별
        - 숫자·기한·빈도 변경 탐지
        - 의미 해석 금지
        - 법적 판단 금지
        
        이 Agent는 “팩트 기반 구조 분석”만 수행하고, 의미 해석은 하지 않는다.
        
        ```bash
        제23조 제2항
        1년 → 6개월
        변경 유형: 빈도 증가
        ```
        
    - **Risk Analyst Agent(법적 책임 관점 전문가)**
        
        입력:
        
        - Change Analyst 결과
        
        역할:
        
        - 의무 강도 변화 분석
        - 처벌 수위 변화 탐지
        - 법적 책임 확대/축소 판단
        - 리스크 등급 산정
        
        이 Agent는 “법적 책임 관점”만 보고, 조직 상황은 고려하지 않는다.
        
        ```bash
        의무 빈도 2배 증가
        불이행 시 과태료 유지
        관리 부담 증가
        리스크 등급: Medium
        ```
        
    - **Control Auditor Agent(조직 내부통제 관점 전문가)**
        
        입력:
        
        - Change 결과
        - 조직 프로파일
        
        역할:
        
        - 적용 대상 여부 판단
        - 기존 내부 통제 체계와 충돌 여부 판단
        - Gap 분석 수행
        - 영향 범위 판단
        
        ```bash
        현재 점검 주기 연 1회
        개정 요구 반기 1회
        통제 미충족
        ```
        
    - **Action Strategist Agent(실행 전략 전문가)**
        
        입력:
        
        - Risk 결과
        - Control Gap 분석
        
        역할:
        
        - 구체적 실행 항목 도출
        - 우선순위 설정
        - 예상 비용 영향 추정
        
        ```bash
        1. 점검 매뉴얼 수정 (High)
        2. 교육 주기 변경 (Medium)
        3. 예산 재산정 필요
        ```
        

### 8. 핵심 기능 정의

**1) 입법예고 유형 자동 분류**

- 일부개정 / 전부개정 / 신설 / 타법개정 자동 판별
- 개정 대상 법령명 및 법령 ID 식별
- 개정 조문 존재 여부 판단
- 부칙 포함 여부 식별
- 처리 전략(Strategy) 자동 선택

**2) 조문 ID 기반 정밀 매칭**

- 개정 대상 조문 번호 추출 (예: 제23조 제2항)
- 법령 구조 DB 내 표준 조문 ID 매핑
- 시행일 기준 최신 현행 조문 자동 선택
- 다중 개정 이력 중 유효 버전 필터링
- 조문 번호 불일치 시 유사 매칭 보조 탐색

**3) 구조 DB 기반 Diff 분석**

- 기존 조문 vs 개정 조문 문장 단위 비교
- 숫자·기한·빈도 변화 자동 탐지
- 삭제/신설 문장 탐지
- 문구 변경 영역 하이라이트
- 변경 유형 1차 분류 (신설/삭제/강화/완화/의무추가 등)

**4) Vector 기반 의미 해석**

- 변경 문장의 의미 유사도 기반 분석
- 의무 강도 변화 추론
- 처벌·벌칙 조항 연계 탐색
- 관리 부담 증가 여부 판단
- RAG 기반 법령 맥락 참조

**5) 조직 영향 분석**

- 조직 프로파일(사업장 유형·설비 보유 여부) 조회
- 적용 대상 조건과 조직 데이터 매칭
- 내부 점검·교육·관리 주기와 비교
- 통제 Gap 여부 판단
- 영향 범위(전사/특정 사업장) 식별

**6) 액션 아이템 도출**

- Gap 기반 구체적 실행 과제 생성
- 우선순위 자동 산정 (리스크 등급 기반)
- 단기/중기/장기 조치 구분
- 예상 관리 비용 증가 추정
- 담당 부서 제안

**7) 경영진 요약 리포트 자동 생성**

- 변경 요약 (핵심 한 줄 요약 포함)
- 리스크 등급 표시
- 조직 영향 여부 명확화
- 대응 필요성 판단 (즉시/모니터링/불필요)
- 의사결정용 Bullet Summary 생성
